Quick Start (30 Minutes)

Get your multi-node WiFi sensing system running in 30 minutes using the automated setup script.

Requirements: 3x ESP32-S3 devices, Orange Pi 5 Pro (or Linux host), WiFi network

1 Clone Repository

git clone https://github.com/prigess/RuView.git
cd RuView
git checkout fix/sensing-improvements

2 Run Automated Setup

./scripts/setup-multi-node.sh

The interactive script will guide you through:

  • Building the sensing server
  • Building ESP32-S3 firmware
  • Flashing each device
  • Provisioning WiFi credentials
  • Starting the server

3 Open the UI

http://<HOST_IP>:8080

You should see all 3 nodes appearing in the dashboard with real-time CSI data.

Data Flow

Component Protocol Role
WiFi Router/AP 802.11 Transmits WiFi frames; body affects channel state
ESP32-S3 Nodes 802.11 (rx), UDP (tx) Capture CSI, send ADR-018 frames to server
Sensing Server UDP :5005, HTTP :8080, WS :8765 Process CSI, run ML pipeline, serve UI/API
Browser/Client HTTP, WebSocket Display real-time sensing results

Detection Thresholds (v0.7.0)

Classification Score Threshold Description
Active > 0.20 Significant movement detected
Present (Moving) > 0.08 Light movement or walking
Present (Still) > 0.025 Stationary person detected
Absent ≤ 0.025 No presence detected

Install Prerequisites (Orange Pi / Ubuntu)

# Update system
sudo apt update && sudo apt upgrade -y

# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# Install Python dependencies
sudo apt install -y python3-pip python3-serial
pip3 install esptool

# Install Docker (optional, for firmware build)
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

Automated Setup

Setup Script

The setup-multi-node.sh script provides an interactive menu for all setup tasks:

./scripts/setup-multi-node.sh

Menu Options

Option Description
1) Full setup Complete end-to-end setup (build, flash, provision, start)
2) Build firmware only Build ESP32-S3 firmware using Docker or native ESP-IDF
3) Flash nodes only Flash firmware to connected ESP32-S3 devices
4) Provision nodes only Configure WiFi credentials on flashed devices
5) Start sensing server Start the server with ESP32 source
6) Identify devices Monitor serial output to identify physical devices

Manual Setup (Step-by-Step)

1 Clone and Build Server

git clone https://github.com/prigess/RuView.git
cd RuView
git checkout fix/sensing-improvements

# Build sensing server
cd v2
cargo build --release -p wifi-densepose-sensing-server

2 Build ESP32-S3 Firmware

cd firmware/esp32-csi-node

# Using Docker (recommended)
docker run --rm -v "$(pwd)":/project -w /project \
  espressif/idf:v5.4 idf.py build

# OR using native ESP-IDF
source /path/to/esp-idf/export.sh
idf.py build

3 Identify ESP32 Ports

# Linux
ls /dev/ttyUSB* /dev/ttyACM*

# macOS
ls /dev/cu.usbserial* /dev/cu.SLAB*

# Windows (Git Bash)
ls /dev/ttyS* | grep -v ttyS0

4 Flash Each Device

# Flash device 1
esptool.py --chip esp32s3 -p /dev/ttyUSB0 -b 460800 \
  --before default_reset --after hard_reset \
  write_flash --flash_mode dio --flash_size 8MB \
  0x0 build/bootloader/bootloader.bin \
  0x8000 build/partition_table/partition-table.bin \
  0x10000 build/esp32-csi-node.bin

# Repeat for /dev/ttyUSB1, /dev/ttyUSB2, etc.

5 Provision WiFi

# Get your server's IP address
hostname -I  # Linux
ipconfig getifaddr en0  # macOS

# Provision each node
python firmware/esp32-csi-node/provision.py \
  --port /dev/ttyUSB0 \
  --ssid "YourWiFi" \
  --password "YourPassword" \
  --target-ip 192.168.1.100:5005

# Repeat for each device

6 Start Server

cd v2
cargo run --release -p wifi-densepose-sensing-server -- \
  --bind-addr 0.0.0.0 \
  --source esp32 \
  --http-port 8080 \
  --udp-port 5005

7 Access UI

Open in browser: http://<SERVER_IP>:8080

Verification

Check Connected Nodes

curl -s http://localhost:8080/api/v1/nodes | python3 -m json.tool

Expected output:

{
  "nodes": [
    {"node_id": 1, "rssi": -45, "last_seen_ms": 100},
    {"node_id": 2, "rssi": -52, "last_seen_ms": 150},
    {"node_id": 3, "rssi": -48, "last_seen_ms": 120}
  ]
}

Check Server Status

curl -s http://localhost:8080/api/v1/status

Monitor Real-Time Stream

# Install websocat if needed: cargo install websocat
websocat ws://localhost:8765/ws/sensing

Monitor ESP32 Serial Output

python -m serial.tools.miniterm /dev/ttyUSB0 115200

Expected output shows CSI frames being sent:

[CSI] Frame 12345: rssi=-45 subs=56 -> 192.168.1.100:5005
[CSI] Frame 12346: rssi=-46 subs=56 -> 192.168.1.100:5005

Troubleshooting

Node Not Appearing in Dashboard

Symptom Cause Solution
Node not in list Wrong target IP Re-provision with correct server IP
Node not in list Firewall blocking UDP 5005 sudo ufw allow 5005/udp
Node not in list WiFi not connected Check serial output for WiFi errors
Node offline No data for 5+ seconds Check node is powered and in range

Poor Detection Accuracy

  • Position nodes 2-5 meters apart for best coverage
  • Avoid placing nodes behind metal objects
  • Run field calibration: curl -X POST http://localhost:8080/api/v1/calibrate
  • Ensure room is empty during calibration (3 minutes)

Build Errors

Error Solution
Docker not found Install Docker or use native ESP-IDF
Rust compilation error rustup update to latest stable
esptool not found pip install esptool
Permission denied on serial sudo usermod -aG dialout $USER (relogin)

Common Serial Port Names

OS Port Pattern
Linux /dev/ttyUSB0, /dev/ttyACM0
macOS /dev/cu.usbserial-*, /dev/cu.SLAB_*
Windows COM3, COM4, etc.

Endpoint Method Description

/api/v1/recording/start POST Start recording CSI data
/api/v1/recording/stop POST Stop recording and save dataset
/health GET Health check for monitoring
